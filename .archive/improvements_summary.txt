   C) QueryCache
      • TTL-based query result caching
      • Default TTL: 5 minutes
      → Reduce DB I/O

📊 Performance Impact:
   • Repeated predictions: 10x faster
   • Expert system inference: 50x faster
   • Typical cache hit rate: 70-80%

TROUBLESHOOTING
════════════════════════════════════════════════════════════════════

Problem: ModuleNotFoundError for new modules
→ Ensure PYTHONPATH includes project root
→ Run scripts from project root: python scripts/train_v2.py

Problem: "database is locked" error
→ Close all connections before migration
→ Ensure no other processes using DB

Problem: Low cache hit rate
→ Increase prediction_cache.max_size
→ Clear cache periodically: engine.clear_cache()

Problem: Model accuracy didn't improve
→ Use larger n_iter in RandomizedSearchCV
→ Try GridSearchCV for exhaustive search
→ Verify data quality and features

Problem: SQLite "no such table" error
→ Run scripts/migrate_to_sqlite.py first
→ Check database file exists and has proper permissions
